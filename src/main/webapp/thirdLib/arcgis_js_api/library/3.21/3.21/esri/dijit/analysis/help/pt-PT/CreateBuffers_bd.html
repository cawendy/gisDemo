Criar Buffers

Um buffer é uma área que abrange uma determinada distância de um elemento de ponto, linha ou área.

Os buffers são normalmente utilizados para criar áreas que podem ser sujeitas a análise adicional utilizando outras ferramentas. Por exemplo, se a pergunta é "Que edifícios se encontram a uma milha da escola?", a resposta pode ser encontrada através da criação de um buffer de 1 milha em redor da escola e sobrepondo o buffer com a camada que contém os limites dos edifícios. O resultado final é uma camada com os edifícios que se encontram a 1 milha da escola.

Selecionar uma camada à qual aplicar buffer

O ponto, linha, ou elementos de área onde aplicar o buffer. As camadas de entrada têm de encontrar-se num sistema de coordenadas projetado que utilize os Ambientes de Análise.

Para além de selecionar uma camada do seu mapa, pode selecionar Explorar Camadas na parte inferior da lista pendente para navegar até aos seus conteúdos de um conjunto de dados de partilha de big data ou camada de elementos.

Selecionar o tipo de buffer a aplicar

Existem três maneiras de especificar o tamanho do buffer para os seus elementos de entrada:

  • Especificar uma distância a aplicar a todos os elementos.
  • Especificar um campo, na camada de entrada, que contenha o valor da distância. Pode utilizar um campo de string ou um campo numérico. Caso não esteja definida uma unidade linear, serão utilizadas as unidades de referência espacial. Caso esteja a utilizar um sistema de coordenadas geográficas, assumir-se-á que o campo que não possui qualquer unidade seja representado em metros.
  • Construir uma expressão com vários campos e operadores matemáticos.

Selecionar o método de buffer

Pode optar por utilizar um método Planar ou um método Geodésico . O método Planar pode ser mais rápido e é apropriado para análise local de dados projetados. O método Geodésico é apropriado para áreas de grande dimensão e para qualquer sistema de coordenadas geográficas.

Selecionar o tipo de método dissolver

Opções para especificar o método dissolver. Se for escolhido um método dissolver, ser-lhe-á dada a opção de criar áreas com várias partes ou únicas, e poderá calcular estatísticas com base nos campos fornecidos.

  • Todos—Todos os elementos serão dissolvidos. Caso sejam permitidos elementos de várias partes, todos os elementos serão dissolvidos num único elemento. Caso não sejam permitidos elementos de várias partes, os elementos adjacentes ou sobrepostos serão dissolvidos.
  • Campos—Elementos que partilham o mesmo campo especificado, ou a mesma combinação de campos especificada, serão dissolvidos. Caso sejam permitidos elementos de várias partes, todos os elementos com o mesmo campo serão dissolvidos num único elemento. Caso não sejam permitidos elementos de várias partes, os elementos adjacentes ou sobrepostos com o mesmo valor de campo serão dissolvidos.
  • Nenhum—não serão dissolvidos quaisquer elementos. Isto é por omissão.

Permitir elementos de várias partes

Opção para especificar se o seu resultado consistirá em elementos únicos ou de várias partes.

  • Marcada—Os resultados da sua análise irão conter elementos de várias partes.
  • Não Marcada— Os resultados da sua análise irão conter elementos únicos. Esta é a predefinição.

Adicionar estatísticas (opcional)

Pode calcular estatísticas em elementos que se encontram resumidos. Em campos numéricos, pode calcular as seguintes:

  • Contagem—Calcula o número de valores não-nulos. Pode ser utilizado em campos numéricos ou em strings. A contagem de [null, 0, 2] é 2.
  • Soma—A soma dos valores numéricos num campo. A soma de [null, null, 3] é 3.
  • Média—A média de valores numéricos. A média de [0, 2, null] é 1.
  • Mín—o valor mínimo de um campo numérico. O mínimo de [0, 2, null] é 0.
  • Máx—o valor máximo de um campo numérico. O valor máximo de [0, 2, null] é 2.
  • Intervalo—o intervalo de um campo numérico. Este é calculado subtraindo os valores mínimos ao valor máximo. O intervalo de [0, null, 1] é 1. O intervalo de [null, 4] é 0.
  • Variância—a variância de um campo numérico num trilho. A variância de [1] é null. A variância de [null, 1,1,1] é 1.
  • Desvio padrão—O desvio padrão de um campo numérico. O desvio padrão de [1] é null. O desvio padrão de [null, 1,1,1] é 1.

Em campos de string, pode calcular as seguintes:

  • Contagem—O número de strings não-nulas.
  • Qualquer—Esta estatística é uma amostra aleatória de um valor de string no campo especificado.
Todas as estatísticas são calculadas em campos que não são nulos. A camada resultante irá conter um novo campo por cada estatística calculada. É possível adicionar qualquer número de estatísticas, selecionando um atributo e uma estatística.

Selecionar um ArcGIS Data Store no qual guardar os resultados

Os resultados de GeoAnalytics são armazenados num ArcGIS Data Store e exibidos como uma camada de elementos no Portal for ArcGIS. Na maioria dos casos, os resultados deverão ser armazenados num armazenamento de dados espaciotemporal, sendo este o comportamento predefinido. Em alguns casos, será uma boa opção guardar os resultados no armazenamento de dados relacional. As seguintes são razões que justificam o armazenamento de resultados no armazenamento de dados espaciotemporal.

  • Utilizar resultados em colaboração portal to portal.
  • Ativar funcionalidades de sincronização com os resultados.

Não deverá utilizar o armazenamento de dados relacional caso antecipe um aumento dos seus resultados de GeoAnalytics, e caso necessite de tirar partido das funcionalidades de armazenamento de big data para gerir grandes quantidades de dados.

Nome da camada resultante

Este é o nome da camada que será criada em O Meu Conteúdo e adicionada ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Caso a camada já existe, a ferramenta irá falhar.

Ao utilizar a caixa suspensa Guardar resultado em pode definir o nome da pasta em O Meu Conteúdo onde o resultado será guardado.
