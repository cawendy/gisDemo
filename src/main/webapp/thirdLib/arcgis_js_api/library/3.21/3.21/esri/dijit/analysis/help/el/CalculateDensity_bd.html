Υπολογισμός Πυκνότητας

The Calculate Density tool creates a density map from point features by spreading known quantities of some phenomenon (represented as attributes of the points) across the map. The result is a layer of areas classified from least dense to most dense.

For point input, each point should represent the location of some event or incident, and the result layer represents a count of the incident per unit area. A higher density value in a new location means that there are more points near that location. In many cases, the result layer can be interpreted as a risk surface for future events. For example, if the input points represent locations of lightning strikes, the result layer can be interpreted as a risk surface for future lightning strikes.

Other use cases of this tool include the following:

  • Creating crime-density maps to help police departments properly allocate resources to high crime areas.
  • Calculating densities of hospitals within a county. The result layer will show areas with high and low accessibility to hospitals, and this information can be used to decide where new hospitals should be built.
  • Identifying areas that are at high risk of forest fires based on historical locations of forest fires.
  • Locating communities that are far from major highways in order to plan where new roads should be constructed.

Διαλέξτε ένα σημειακό θεματικό επίπεδο από το οποίο θα υπολογιστεί η πυκνότητα

The point layer on which the density will be calculated.

Εκτός από την επιλογή ενός θεματικού επιπέδου από το χάρτη σας, μπορείτε επίσης να επιλέξετε Αναζήτηση θεματικών επιπέδων στο κάτω μέρος της αναπτυσσόμενης λίστας για αναζήτηση στα περιεχόμενά σας ενός κοινόχρηστου αρχείου συνόλου δεδομένων με big data ή ενός feature layer.

Διαλέξτε ένα ή περισσότερα πεδία όπου θα υπολογιστεί η πυκνότητα (προαιρετικό)

Provide one or more fields specifying the number of incidents at each location. For example, if you have points that represent cities, you can use a field representing the population of the city as the count field, and the resulting population density layer will calculate larger population densities near cities with larger populations.

The density for the count of points will always be calculated. If no fields are selected, each location will be assumed to represent a single count.

Διαλέξτε ένα σχήμα bin

The shape of bins that points are aggregated into and used for the density calculation.

Για την ανάλυση με χρήση τετράγωνων ή εξάγωνων bin, απαιτείται προβολικό σύστημα συντεταγμένων. Μπορείτε να ρυθμίσετε το Σύστημα συντεταγμένων επεξεργασίας στα Περιβάλλοντα ανάλυσης. Εάν το σύστημα συντεταγμένων επεξεργασίας δεν έχει οριστεί σε προβολικό σύστημα συντεταγμένων, θα σας ζητηθεί να το ορίσετε κατά την Εκτέλεση ανάλυσης.

Επιλέξτε το μέγεθος bin για τη συνάθροιση

The size of the bins generated. For a square bin, the size represents the height of the square. For hexagon, the size represents the height of the hexagon (from parallel sides).

Επιλέξτε τη στάθμιση πυκνότητας προς εφαρμογή

The type of weighting applied to the density calculation. There are two options:

  • Uniform—This calculates a magnitude per area.
  • Kernel—Applies a kernel function to fit a smooth tapered surface to each point.

Επιλέξτε το μέγεθος γειτονιάς

The size of the area used to calculate the density. The neighborhood size must be larger than the bin size.

Διαλέξτε τις μονάδες εξόδου για την πυκνότητα

The desired output units of the density values. By default, this is set to square kilometers or square miles based on the default units specified in your profile. If density values are very small, you can increase the size of the area units (for example, square meters to square kilometers) to return larger values. The values only scale the result.

Υπολογισμός πυκνότητας με χρονικά βήματα (προαιρετικά)

Αν είναι ενεργοποιημένος ο χρόνος στο σημειακό θεματικό επίπεδο εισόδου, και ως τύπος έχει επιλεχθεί στιγμή, η ανάλυση μπορεί να γίνει χρησιμοποιώντας χρονική κατάτμηση. Υπάρχουν τρεις παράμετροι που μπορείτε να ορίσετε με χρήση του χρόνου:

  • Διάστημα χρονικού βήματος
  • Συχνότητα επανάληψης χρονικού βήματος
  • Χρόνος για ευθυγράμμιση των χρονικών βημάτων

Φανταστείτε ότι έχετε δεδομένα που αναπαριστούν χρονικά ένα έτος και θέλετε να κάνετε ανάλυση χρησιμοποιώντας εβδομαδιαία χρονικά βήματα. Σε αυτή την περίπτωση, πρέπει να ορίσετε το Διάστημα του χρονικού βήματος σε 1 εβδομάδα.

Φανταστείτε ότι έχετε δεδομένα που αναπαριστούν χρονικά ένα έτος και θέλετε να κάνετε ανάλυση χρησιμοποιώντας την πρώτη εβδομάδα του μήνα. Σε αυτή την περίπτωση, πρέπει να ορίσετε το Διάστημα χρονικού βήματος σε 1 εβδομάδα, τη Συχνότητα επανάληψης χρονικού βήματος σε 1 μήνα, και τον Χρόνο για ευθυγράμμιση των χρονικών βημάτων σε 1 Ιανουαρίου, 12:00 πμ.

Διάστημα χρονικού βήματος

Χρονικό διάστημα για τη δημιουργία των χρονικών βημάτων. Το διάστημα χρονικού βήματος μπορεί να χρησιμοποιηθεί μόνο του ή με τις παραμέτρους Συχνότητα επανάληψης χρονικού βήματος ή Χρόνος για ευθυγράμμιση των χρονικών βημάτων.

Αν θέλατε να δημιουργήσετε χρονικά τμήματα που λαμβάνουν χώρα κάθε Δευτέρα από τις 9:00 μέχρι τις 10:00 π.μ., το Διάστημα χρονικού βήματος θα έπρεπε να είναι 1 ώρα, η Συχνότητα επανάληψης χρονικού βήματος θα έπρεπε να είναι 1 εβδομάδα και ο Χρόνος για ευθυγράμμιση των χρονικών βημάτων θα έπρεπε να είναι 9:00:00 π.μ. τη Δευτέρα.

Συχνότητα επανάληψης χρονικού βήματος

Το βήμα που χρησιμοποιείται για τον υπολογισμό ενός χρονικού τμήματος. Η Συχνότητα επανάληψης χρονικού βήματος μπορεί να χρησιμοποιηθεί μόνη της ή με τις παραμέτρους Διάστημα χρονικού βήματος και Χρόνος αναφοράς ή με τις δύο παραμέτρους μαζί Διάστημα χρονικού βήματος και Χρόνος για ευθυγράμμιση των χρονικών βημάτων.

Αν θέλατε να δημιουργήσετε χρονικά τμήματα που λαμβάνουν χώρα κάθε Δευτέρα από τις 9:00 μέχρι τις 10:00 π.μ., το Διάστημα χρονικού βήματος θα έπρεπε να είναι 1 ώρα, η Συχνότητα επανάληψης χρονικού βήματος θα έπρεπε να είναι 1 εβδομάδα και ο Χρόνος για ευθυγράμμιση των χρονικών βημάτων θα έπρεπε να είναι 9:00:00 π.μ. τη Δευτέρα.

Χρόνος για ευθυγράμμιση των χρονικών βημάτων

Η ημερομηνία και η ώρα που χρησιμοποιούνται για ευθυγράμμιση της χρονικής κατάτμησης. Η χρονική κατάτμηση θα εκκινήσει και θα συνεχίσει αντίστροφα από αυτήν την ώρα. Αν δεν έχει επιλεγεί χρόνος αναφοράς, η χρονική κατάτμηση θα ευθυγραμμιστεί στην 1η Ιανουαρίου 1970.

Αν θέλατε να δημιουργήσετε χρονικά τμήματα που λαμβάνουν χώρα κάθε Δευτέρα από τις 9:00 μέχρι τις 10:00 π.μ., το Διάστημα χρονικού βήματος θα έπρεπε να είναι 1 ώρα, η Συχνότητα επανάληψης χρονικού βήματος θα έπρεπε να είναι 1 εβδομάδα και ο Χρόνος για ευθυγράμμιση των χρονικών βημάτων θα έπρεπε να είναι 9:00:00 π.μ. τη Δευτέρα.

Χωρική αναφορά (wkid)

Είναι μια προσωρινή παράμετρος για την έκδοση prerelease που ορίζει τη χωρική αναφορά της επεξεργασίας. Πολλά εργαλεία big data απαιτούν τη χρήση ενός προβολικού συστήματος συντεταγμένων ως τη χωρική αναφορά της επεξεργασίας. Από προεπιλογή, το εργαλείο θα χρησιμοποιεί το σύστημα συντεταγμένων εισόδου αλλά θα αποτύχει εάν πρόκειται για σύστημα γεωγραφικών συντεταγμένων. Για να ορίσετε ένα προβολικό σύστημα συντεταγμένων, εισαγάγετε το WKID. Για παράδειγμα, για το Web Mercator πρέπει να εισαγάγετε 3857.

Επιλέξτε χώρο αποθήκευσης δεδομένων

Τα αποτελέσματα GeoAnalytics αποθηκεύονται σε ένα ArcGIS Data Store και εκτίθενται ως ένα θεματικό επίπεδο στοιχείων στο Portal for ArcGIS. Στις περισσότερες περιπτώσεις, τα αποτελέσματα πρέπει να αποθηκεύονται στο χωρο-χρονικό data store και αυτό αποτελεί την προεπιλογή. Σε μερικές περιπτώσεις, η αποθήκευση αποτελεσμάτων σε σχεσιακό data store είναι μια καλή επιλογή. Οι ακόλουθοι είναι κάποιοι λόγοι για τους οποίους θα αποθηκεύατε τα αποτελέσματα σε σχεσιακό data store:

  • Χρήση αποτελεσμάτων σε portal για συνεργασία portal.
  • Ενεργοποίηση δυνατοτήτων συγχρονισμού με τα αποτελέσματά σας.

Δεν πρέπει να χρησιμοποιείτε το σχεσιακό data store αν περιμένετε αύξηση των αποτελεσμάτων GeoAnalytics και επιθυμείτε να εκμεταλλευτείτε τις δυνατότητες του χωρο-χρονικού big data store για διαχείριση μεγάλων όγκων δεδομένων.
