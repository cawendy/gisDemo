Criar Buffers

O buffer é uma área que cobre uma determinada distância a partir de uma feição de ponto, linha ou área.

Os buffers são normalmente utilizados para criar áreas que podem ser também analisadas utilizando outras ferramentas. Por exemplo, se a pergunta for Quais edifícios estão dentro de 1 milha da escola?, a resposta poderá ser encontrada ao criar um buffer de 1 milha em torno da escola e sobrepor o buffer com a camada contendo as áreas de cobertura dos edifícios. O resultado final é uma camada destes edifícios dentro de 1 milha da escola.

Escolher camada para aplicar o buffer

As feições de ponto, linha ou área a serem geradas com o buffer. As camadas de entrada devem estar em um sistema de coordenadas planas ou a referência espacial de processamento deve ser definida para um sistema de coordenadas planas utilizando os Ambientes de Análise.

Além de escolher uma camada de seu mapa, você pode escolher Procurar Camadas na parte inferior da lista suspensa para procurar por seu conteúdo para um conjunto de dados de compartilhamento de arquivo big data ou camada de feição.

Selecione o tipo de buffer para aplicar

Há três maneiras que você pode especificar o tamanho do buffer para suas feições de entrada:

  • Especificar uma distância que é aplicada para todas as feições.
  • Especificar um campo na camada de entrada que contém o valor de distância. Você pode utilizar um campo de string ou um campo numérico. Se uma unidade linear não estiver definida, as unidades da referência espacial serão utilizadas. Se você estiver utilizando um sistema de coordenadas geográficas, o campo sem unidade será assumido como sendo metros.
  • Construir uma expressão com múltiplos campos e operadores matemáticos.

Selecione o método de buffer

Você pode escolher para utilizar o método Planar ou Geodésico. O método Planar pode ser mais rápido e é apropriado para análise local em dados projetados. O método Geodésico é apropriado para grandes áreas e qualquer sistema de coordenadas geográficas.

Selecione o tipo de método para dissolver

Opções para especificar o método dissolver. Se o método dissolver for escolhido, você receberá a opção de criar áreas de parte única ou de múltiplas partes e você poderá calcular a estatística baseado em campos fornecidos.

  • Todos—Todas as feições serão dissolvidas. Se feições de múltiplas partes forem permitidas, todas as feições serão dissolvidas em uma única feição. Se múltiplas partes não forem permitidas, as feições que estiverem adjacentes ou sobrepondo serão dissolvidas.
  • Campos—As feições que compartilham o mesmo campo especificado ou combinação de campo especificado serão dissolvidas. Se feições de múltiplas partes forem permitidas, todas as feições com o mesmo campo serão dissolvidas em uma única feição. Se múltiplas partes não forem permitidas, as feições que são adjacentes ou sobrepondo com o mesmo valor de campo serão dissolvidas.
  • Nenhum—Nenhuma feição será dissolvida. Este é o padrão.

Permitir feições de múltiplas partes

Opção para especificar se o seu resultado consistirá em feições de única parte ou de múltiplas partes.

  • Marcado—Os resultados da sua análise estarão contidos em feições de múltiplas partes.
  • Desmarcado—Os resultados da sua análise estarão contidos em feições de parte única. Este é o padrão.

Adicionar estatísticas (opcional)

Você pode calcular as estatística em feições que são resumidas. Nos campos numéricos você pode calcular o seguinte:

  • Contagem—Calcula o número de valores diferentes de nulo. Pode ser utilizado em campos numéricos ou strings. 'A contagem de [null, 0, 2] é 2.
  • Soma—A soma de valores numéricos em um campo. A soma de [null, null, 3] é 3.
  • Média—A média de valores numéricos. A média de [0, 2, null] é 1.
  • Mín—O valor mínimo de um campo numérico. O mínimo de [0, 2, null] é 0.
  • Máx—O valor máximo de um campo numérico. O valor máximo de [0, 2, null] é 2.
  • Alcance—O alcance de um campo numérico. Isto é calculado como os valores mínimos subtraídos do valor máximo. O alcance de [0, null, 1] é 1. O alcance de [null, 4] é 0.
  • Variância—A variância de um campo numérico em um caminho. A variância de [1] é nula. A variância de [null, 1,1,1] é 1.
  • Desvio padrão—O desvio padrão de um campo numérico. O desvio padrão de [1] é nulo. O desvio padrão de [nulo, 1,1,1] é 1.

Nos campos de string você pode calcular o seguinte:

  • Contagem—O número de strings diferente de nulo.
  • Qualquer—Esta estatística é uma amostra aleatória de um valor de string no campo especificado.
Todas as estatísticas são calculadas em valores não nulos. A camada resultante conterá um novo atributo para cada estatística calculada. Qualquer número de estatística pode ser adicionado ao escolher um atributo e estatística.

Escolha um ArcGIS Data Store para salvar os resultados

Os resultados do GeoAnalytic são armazenados no ArcGIS Data Store e expostos como uma camada de feição no Portal for ArcGIS. Na maioria dos casos, os resultados devem ser armazenados no armazenamento de dados de espaço-tempo e este é o padrão. Em alguns casos salvar resultados no armazenamento de dados relacional é uma boa opção. Abaixo estão as razões pelas quais você talvez queira armazenar resultados em um armazenamento de dados relacional:

  • Utilizar os resultados no portal para colaboração do portal.
  • Habilitar recursos de sincronização com seus resultados.

Você não deve utilizar um armazenamento de dados relacional se você espera que os resultados do GeoAnalytics aumentem e deseja aproveitar os recursos do armazenamento de grandes dados de espaço-tempo para lidar com grandes quantidades de dados.

Nome da camada resultante

Este é o nome da camada que será criada em Meu Conteúdo e adicionada no mapa. O nome padrão é baseado no nome da ferramenta e o nome da camada de entrada. Se a camada já existe, a ferramenta falhará.

Ao utilizar a caixa suspensa Salvar resultado em, é possível especificar o nome de uma pasta em Meu Conteúdo onde o resultado será salvo.
