สร้างพื้นที่ขยาย

ระยะกันชน คือ พื้นที่ที่ครอบคลุมทั้งข้อมูลจุด เส้น หรือพื้นที่ที่สนใจ โดยมีระยะทางที่ขยายออกไปเท่าๆ กัน

ระยะกันชนมักจะใช้ในการสร้างพื้นที่ที่สามารถวิเคราะห์เพิ่มเติมได้โดยใช้เครื่องมืออื่น ๆ ตัวอย่างเช่น, ถ้าคำถามคือสิ่งที่อยู่ภายในอาคาร 1 ไมล์ของโรงเรียน ?, คำตอบที่สามารถพบได้โดยการสร้างบัฟเฟอร์ 1 ไมล์รอบ ๆ โรงเรียนและซ้อนทับบัฟเฟอร์ด้วยชั้นข้อมูลที่มีสิ่งปลูกสร้าง ผลลัพธ์ที่ได้คือชั้นข้อมูลของอาคารเหล่านั้นภายใน 1 ไมล์ของโรงเรียน

เลือกชั้นข้อมูลที่จะใช้ในการบัฟเฟอร์

ฟีเจอร์ของจุด เส้น หรือพื้นที่ที่จะสร้างแนวกันชน ชั้นข้อมูลอินพุทต้องอยู่ในระบบพิกัดโปรเจคชัน หรือกำหนดผลการวิเคราะห์เป็นระบบพิกัดโปรเจคชัน โดยกำหนดด้วย สภาพแวดล้อมการวิเคราะห์

นอกเหนือจากการเลือกชั้นข้อมูลจากแผนที่คุณสามารถเลือก เบราว์เลเยอร์ ที่ด้านล่างของรายการแบบเลื่อนลงเพื่อเรียกดูเนื้อหาของคุณสำหรับชุดข้อมูลชุดข้อมูลข้อมูลขนาดใหญ่หรือฟีเจอร์เลเยอร์

เลือกประเภทของบัฟเฟอร์ที่จะนำไปใช้

มีสามวิธีที่คุณสามารถระบุขนาดของบัฟเฟอร์สำหรับข้อมูลอินพุทของคุณ:

  • กำหนดระยะทางให้กับฟีเจอร์ทั้งหมด
  • ระบุฟิลด์ให้กับชั้นข้อมูลอินพุทนั้นจะต้องประกอบด้วยค่าระยะทาง คุณสามารถใช้ฟิลด์ตัวอักษรหรือฟิลด์ตัวเลข หากหน่วยเชิงเส้นไม่ได้ถูกกำหนด, หน่วยของการอ้างอิงเชิงพื้นที่จะถูกนำมาใช้ ถ้าคุณใช้ระบบพิกัดภูมิศาสตร์, จะไม่แสดงหน่วยและจะถูกกำหนดให้เป็นหน่วยเมตร
  • สร้างเงื่อนไขที่มีหลายฟิลด์และดำเนินการทางคณิตศาสตร์

เลือกวิธีการบัฟเฟอร์

คุณสามารถเลือกใช้ ระนาบ หรือวิธีการ พื้นผิวหน้าโค้ง วิธีการ ส่วน ระนาบ วิธีการนี้อาจจะเร็วและเหมาะสำหรับการวิเคราะห์ของข้อมูลที่คาดการณ์ไว้ ส่วน พื้นผิหน้าโค้ง วิธีการนี้เหมาะสำหรับพื้นที่ใหญ่และใช้ระบบพิกัดภูมิศาสตร์

เลือกประเภทของวิธีการรวม

ตัวเลือกเพื่อกำหนดวิธีการรวม ถ้าวิธีการรวมถูกเลือก, คุณจะได้รับตัวเลือกของการสร้างพื้นที่ multipart หรือ single-part และคุณสามารถที่จะคำนวณค่าสถิติที่อยู่ในฟิลด์ที่ให้ไว้ได้

  • ทั้งหมด—ทุกฟีเจอร์จะถูกรวม ถ้าเป็นมัลติพาร์ทฟีเจอร์, ทุกฟีเจอร์จะถูกรวมเข้าไว้ด้วยกันเป็นฟีเจอร์เดียว ถ้าไม่ใช่มัลติพาร์ทฟีเจอร์, ฟีเจอร์ที่อยู่ติดกันหรือทับซ้อนกันจะถูกยุบ
  • ฟิลด์—ฟีเจอร์ที่ใช้ฟิลด์เดียวกันที่กำหนดหรือรวมข้อมูลที่ระบุไว้จะถูกยุบ ถ้าเป็นมัลติพาร์ทฟีเจอร์, ทุกฟีเจอร์กับฟิลด์ที่เหมือนกันจะถูกรวมเข้าไว้ด้วยกันเป็นฟีเจอร์เดียว ถ้าไม่ใช่มัลติพาร์ทฟีเจอร์, ฟีเจอร์ที่อยู่ติดกันหรือทับซ้อนกันกับฟิลด์ที่มีค่าเหมือนกันจะถูกยุบ
  • ไม่เลือก—ฟีเจอร์จะไม่ถูกรวม นี้คือค่าตั้งต้น

อนุญาตสำหรับข้อมูลมัลติพาร์ท

ตัวเลือกที่จะกำหนดว่าผลลัพธ์ของคุณจะประกอบด้วย single-part หรือ multipart features

  • เช็ค—ผลลัพธ์การวิเคราะห์ของคุณจะประกอบด้วย multipart features.
  • ไม่เช็ค—ผลลัพธ์การวิเคราะห์ของคุณจะประกอบด้วย single-part features, นี้คือค่าตั้งต้น

เพิ่มสถิติ (อุปกรณ์เสริม)

คุณสามารถคำนวณสถิติเกี่ยวกับคุณสมบัติที่มีรายละเอียด ในช่องตัวเลขคุณสามารถคำนวณค่า ต่อไปนี้:

  • นับ - นับจำนวนข้อมูลที่ไม่ใช่ค่าว่าง สามารถได้ทั้งในฟิลด์ตัวเลข หรือตัวอักษร นับจำนวนของ [null, 0, 2] คือ 2
  • รวม - ผลรวมของค่าตัวเลขในฟิลด์ ผลรวมของ [null, null, 3] คือ 3
  • เฉลี่ย - ค่าเฉลี่ยของค่าตัวเลข ค่าเฉลี่ยของ [0, 2, null] เท่ากับ 1
  • ต่ำสุด - ค่าต่ำสุดของฟิลด์ตัวเลข ค่าต่ำสุดของ [0, 2, null] เท่ากับ 0
  • สูงสุด - ค่าสูงสุดของฟิลด์ตัวเลข ค่าสูงสุดของ [0, 2, null] คือ 2
  • ช่วง - ช่วงของฟิลด์ตัวเลข ค่านี้คำนวณจากค่าสูงสุด ลบด้วยค่าต่ำสุด ช่วงของ [0, null, 1] คือ 1 ช่วงของ [null, 4] คือ 0
  • ความแปรปรวน - ความแปรปรวนของฟิลด์ตัวเลขในแทร็ก ความแปรปรวนของ [1] เป็น null ความแปรปรวนของ [null, 1,1,1] เท่ากับ 1
  • ค่าเบี่ยงเบนมาตรฐาน - ค่าเบี่ยงเบนมาตรฐานของฟิลด์ตัวเลข ค่าเบี่ยงเบนมาตรฐานของ [1] เป็น null ค่าเบี่ยงเบนมาตรฐานของ [null, 1,1,1] เท่ากับ 1

ในช่องสตริงคุณสามารถคำนวณต่อไปนี้:

  • นับ – นับจำนวนข้อมูลสตริงที่ไม่ใช่ค่าว่าง
  • ค่าใดๆ ต่อไปนี้ เป็นสถิติตัวอย่างที่สุ่มจากค่าสตริงในฟิลด์ที่ระบุ
สถิติทั้งหมดจะมีการคำนวณค่า nonnull ชั้นข้อมูลที่เกิดขึ้นจะมีข้อมูลใหม่สำหรับแต่ละสถิติคำนวณ จำนวนสถิติใด ๆ ที่สามารถเพิ่มได้โดยการเลือกคุณลักษณะและสถิติ

เลือกที่เก็บข้อมูล ArcGIS ดาต้าสโตร์ เพื่อบันทึกผลลัพธ์

GeoAnalytic ผลลัพธ์จะถูกเก็บไว้ใน ArcGIS Data Store และถูกสร้างเป็นชั้นข้อมูลฟีเจอร์ ใน Portal for ArcGIS ในกรณีส่วนใหญ่ ผลลัพธ์ควรเก็บไว้ในที่เก็บข้อมูล spatiotemporal และใช้เป็นค่าเริ่มต้นด้วย ในบางกรณีการบันทึกผลลัพธ์ ไปยังที่เก็บข้อมูลเชิงสัมพันธ์เป็นตัวเลือกที่ดี ต่อไปนี้คือเหตุผลที่คุณอาจต้องการจัดเก็บผลลัพธ์ไว้ ในการเก็บข้อมูลเชิงสัมพันธ์:

  • มีการใช้งานผลลัพธ์ในพอร์ทัล เพื่อใช้กับพอร์ทัลแบบทำงานร่วมกัน
  • เปิดใช้งานความสามารถในการซิงค์กับผลลัพธ์ของคุณ

คุณไม่ควรใช้ที่เก็บข้อมูลเชิงสัมพันธ์ ถ้าคุณคาดว่าจะนำผลลัพธ์จาก GeoAnalytics ของคุณไปใช้ประโยชน์เพิ่มขึ้นจากความสามารถในการจัดเก็บข้อมูลขนาดใหญ่ของ spatiotemporal เพื่อจัดการกับข้อมูลจำนวนมาก

ชื่อของชั้นข้อมูลผลลัพธ์

เป็นชื่อชั้นข้อมูลที่จะถูกสร้างใน เนื้อหาของฉัน และเพิ่มเข้าไปในแผนที่ ชื่อเริ่มต้นจะขึ้นอยู่กับชื่อเครื่องมือและชื่อชั้นข้อมูลอินพุท ถ้าชั้นข้อมูลมีอยู่แล้วเครื่องมือจะล้มเหลว

ใช้ บันทึกผลลัพธ์ใน รายการตัวเลือก เพื่อระบุชื่อของโฟล์เดอร์ใน เนื้อหาของฉัน เมื่อบันทึกผล
